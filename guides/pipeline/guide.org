This is a guide that explains how to run the epimight pipeline to estimates genetic
correlation between two disorders stratified by birth year.

- Any data shown in this guide comes from [[../data/pipeline-tte.csv][this randomly generated time-to-event dataset]]
  - 100 000 individuals
  - 2 disorders (schizophrenia and coronary artery disease)
  - 2 types of relatives (parent/offspring and full siblings)
- All code shown in this guide have been written to [[./guide.R][this R file]]

The basic concept of the pipeline is pretty simple, you create a dataset that contains
time-to-event for all disorders and types of relatives you want to analyse. You feed this
dataset to the pipeline when you create it. Then you tell the pipeline which pair of disorders
and type of relatives you want to analyse and it will output cumulative incidences, heritabilities
and genetic correlations.

The most work you have to do is to format the data in a way that the pipeline expects.

* Glossary

- ~d1~ :: Disorder 1
- ~d2~ :: Disorder 2
- ~c1~ :: Cohort 1, target cohort
- ~c2~ :: Cohort 2, individuals with at least 1 relative diagnosed with ~d1~
- ~c3~ :: Cohort 3, individuals with at least 1 relative diagnosed with ~d2~
- ~cif~ :: Cumulative incidence function
- ~h2~ :: Heritability
- ~rg~ :: Genetic correlation

* Setting up the pipeline

To setup the pipeline, we start by importing the epimight library:

#+BEGIN_SRC R :exports code :eval no
library(dplyr)
library(readr)
library(data.table)
library(epimight)
#+END_SRC

#+BEGIN_SRC R :exports none :eval no
tte <- read_csv(
  "./test-tte.csv",
  show_col_type = FALSE,
  col_types = cols(person_id = col_character()),
) |> as.data.table()
#+END_SRC

And then create a new instance with our TTE pool (the variable ~tte~ is the data table shown in the previous chapter):

#+BEGIN_SRC R :exports both
pipeline <- Pipeline$new(pool = tte)
#+END_SRC

#+RESULTS:

The supplied pool must be a ~data.table~ with the following required columns:

- ~person_id~
  - Must be a string
- ~disorder~
  - Must be a string
- ~failure_status~
  - Must be one of these integers:
    - ~0~ (censored)
    - ~1~ (event of interest)
    - ~2~ (competing risk)
- ~failure_time~
  - Must be numeric, minimum ~0.0~
- ~relationship_kind~
  - Must be one of these strings: [[../data/relationship-kinds.org][names(epimight:::relationship_kinds)]]
- ~relatives~
  - Must be an integer, minimum ~0~
- ~relatives_diagnosed~
  - Must be an integer, minimum ~0~

It accepts any additional columns used for stratification.

* Running the pipeline

After creating the pipeline instance, we run analyses by supplying the ~run~ function with:

- Disorder 1
  - Identifier
  - Earliest onset
  - Latest onset
- Disorder 2
  - Identifier
  - Earliest onset
  - Latest onset
- Relationship kind
- What stratification columns to use (if any)

For example, if we want to run an analysis between schizophrenia and coronary artery disease of full siblings,
stratifying on birth year, we would run:

#+BEGIN_SRC R :exports code :results output
results <- pipeline$run(
  disorder1 = list(
    id             = "SCZ",
    earliest_onset = 1,
    latest_onset   = 100
  ),
  disorder2 = list(
    id             = "CAD",
    earliest_onset = 1,
    latest_onset   = 100
  ),
  relationship_kind = "FS",
  stratify_columns = list("born_at_year")
)
#+END_SRC

#+RESULTS:

This will give us the genetic correlation between the two disorders and all the intermediate results (cif and h2)
in a named list with the members ~cif~, ~h2~ and ~rg~ that contain the results as ~data.tables~.
