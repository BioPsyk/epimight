#+TITLE: Heritability
#+AUTHOR: Richard Zetterberg <richard.zetterberg@regionh.dk>

Heritability is produced by the [[https://r6.r-lib.org/articles/Introduction.html][R6 class]] ~HeritabilityAnalysis~ included in this package.
You can learn how to use this class, either by reading the reference documentation below or read
[[./guide.org][this guide that shows you how it is used, step-by-step with code examples.]]

* Functional reference

To use the class, you need to create an instance first:

#+BEGIN_SRC R
analysis <- HeritabilityAnalysis$new()
#+END_SRC

That instance exposes 2 functions:

- run :: Produces the heritability estimates from the cumulative incidences of two cohorts
- run_meta :: Meta analyses stratified heritability estimates, outputs fixed and random model results

** Using function ~run~

This function estimates the heritability of a single disorder for a specific kind of relationship. It's used to
answer research questions, such as "What is the heritability of schizophrenia in full siblings?".
Other kinds of relationships are parent/offspring, half siblings, grand parent/grand child, etc.

In order to estimate the heritability, this function needs to have two separate cumulative incidence estimates
produced from two cohorts:

- Everyone from the sample space
- People from the sample space, with at least one affected relative

On top of the cumulative incidence estimates, it also needs to know the relationship coefficient of the
kind of relatives being studied.

This function accepts 2 named arguments:

- relationship_kind (required) :: The relationship kind between the sample and their relatives (full sibling, parent-offspring, etc.)
- estimates (required) :: A ~data.table~ of the cumulative incidence estimates for the two cohorts being studied, combined into a single data set,
  that must contain the following columns:

  |---------------------+-----------------------------------------------------------------------|
  | Column name         | Description                                                           |
  |---------------------+-----------------------------------------------------------------------|
  | *time*              | Natural number, failure time in years                                 |
  | *cohort1_estimates* | Floating point number, cohort 1 risk estimates for specific year      |
  | *cohort1_cases*     | Natural number, cohort 1 cumulative amount of cases for specific year |
  | *cohort2_estimates* | Floating point number, cohort 2 risk estimates for specific year      |
  | *cohort2_cases*     | Natural number, cohort 2 cumulative amount of cases for specific year |
  |---------------------+-----------------------------------------------------------------------|

  This ~data.table~ is allowed to contain any number of extra columns, such as when the reults are stratified and there are
  columns that contains the different group values. An example of this is when the estimates are stratified by
  birth year, then you would have a column that contains the birth years. Those extra columns are untouched and will be
  kept in the results.

** Using function ~run_meta~

This function accepts 1 argument:

- results (required) :: A ~data.table~ that contains the results output by the ~run~ function, that must contain the following columns:

  |-------------+----------------------------------------------------------------|
  | Column name | Description                                                    |
  |-------------+----------------------------------------------------------------|
  | *h2*        | Floating point number, heritability estimate                   |
  | *se*        | Floating point number, standard error of heritability estimate |
  |-------------+----------------------------------------------------------------|

  This ~data.table~ is allowed to contain any number of extra columns, but those extra columns will be dropped from the results.
