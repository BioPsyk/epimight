This is a guide for estimating heritability of a single disorder using TTE (time-to-event) data.

All code shown in this guide have been written to [[./guide.R][this R file]], so that you don't have
to copy all code snippets manually.

This guide is similar to the [[../cumulative-incidence/guide.org]["estimate cumulative incidence of disorder"]]-guide.
The difference is that the TTE data also contains how many relatives (of the specific kind)
each individual have and how many of those relatives are also affected by the disorder.
It then constructs a second cohort, which contains all individuals from the target cohort
that have at least 1 affected relative. The heritability estimate is then
derived from comparing the risk estimates from these two cohorts.

The diagram below shows how all data and results are derived, with a single
origin, the TTE data for cohort 1.

In this context, "relatives" refers to relatives of one specific kind of
genealogical relationship. Here are some examples of available relationships:

- Parent/offspring
- Full siblings
- Half siblings
- Grand parent/grand child

That means that this guide has 4 things to configure:

- Study period
- Disorder to study
- Cohort to estimate the risk in
- Relatives relationship kind

We'll use the same research question as the [[../cumulative-incidence/guide.org]["estimate cumulative incidence of disorder"]]-guide,
but we'll extend it to be: "What is the heritability of schizophrenia in Danish millenials
and their full siblings (people born from 1981 to 1996)?"

We start by importing all the needed libraries:

* Arguments

** Study period

The first thing we need to do is to define the arguments that reflects the research question.
Since we want to study millenials, we will use the millenials start year as the start year of
the study:

#+BEGIN_SRC R :exports code
# Arguments for research question:
# What is the lifetime risk of schizophrenia in millenials (people born from 1981 to 1996)?

study_period_start <- as.Date("1981-01-01")
study_period_end   <- as.Date("2016-12-31")
#+END_SRC

#+RESULTS:
: 2016-12-31

The reason why we use ~2016-12-31~ as the end date, is because the registry data in the
~ibp-registry~ database ends at that date.

** Disorder to study

We'll use the same age of onset as the previous guide:

#+BEGIN_SRC R :exports code
earliest_onset_age <- 10
#+END_SRC

#+RESULTS:
: 10

** Cohort to estimate the risk in

Next we define the millenials period (1981 to 1996) as the allowed date of birth:

born_at_start <- as.Date("1981-01-01")
born_at_end   <- as.Date("1996-01-01")
#+END_SRC

#+RESULTS:

** Relatives relationship kind

The library uses a list of codes that represents the different relationship kinds.
Here's a list of the most common ones (there are 30 in total):

|------+--------------------------|
| Code | Description              |
|------+--------------------------|
| PO   | Parent/offspring         |
| 1G   | Grand parent/grand child |
| FS   | Full siblings            |
| HS   | Half siblings            |
| Av   | Avuncular                |
| 1C   | First cousin             |
|------+--------------------------|

We are going to use "full siblings" so we define ~FS~ as our value:

#+BEGIN_SRC R :exports code
relationship_kind <- "FS"
#+END_SRC

#+RESULTS:
: FS

* Data processing

Now that we have defined all arguments to use, it's time to read and process the
TTE data that we are going to analyze. The TTE data needs to fulfill the same
requirements as the [[../cumulative-incidence/guide.org][previous guide]], but also these requirements:

The survival data needs
to fulfill the following requirements in order to be used by this guide:

5. These additional columns *must* also exist:
   - relatives :: Non-negative integer that represents the amount of relatives the individual have
   - diagnosed_relatives :: Non-negative integer that represents how many relatives are diagnosed with the disorder

This time we also need the ~born_at~ column with the birth dates in order to filter out millenials.

The example data in this guide is contained in a file called ~tte_SCZ_FS.csv~,
so we simply read it and convert it into a ~data.table~:

#+BEGIN_SRC R :eval no :exports code
tte <- read_csv("./tte_SCZ_FS.csv") |> as.data.table()
#+END_SRC

#+BEGIN_SRC R :exports results :results output :tangle no
set.seed(6)
n_count <- 100000

tte <- generate_diagnosed_relatives(tte, "diagnosed_relatives") |>
  select(-born_at_year, -dead_at_year) |>

#+RESULTS:

The example data in this guide contains the same 100000 randomly generated in the [[../cumulative-incidence/guide.org][previous guide]],
with the addition of:

- Amount of relatives sampled with the following probablities (0-9): ~c(0.2, 0.22, 0.23, 0.16, 0.09, 0.05, 0.03, 0.01, 0.008, 0.002)~
- If the individual is diagnosed, the following probablities of ~diagnosed_relatives~:
  - No diagnosed relatives :: ~0.45~
  - Probablity per relative count :: ~(1 - 0.55) / relatives~
- If the individual is unaffected, the following probablities of ~diagnosed_relatives~:
  - No diagnosed relatives :: ~0.75~
  - Probablity per relative count :: ~(1 - 0.55) / relatives~

* Analysis

** Risk estimates

Now that we have the survival data for schizophrenia in millenials, we can run the
actual analysis. Since we are going to produce cumulative incidence risk estimates,
we will use the class ~CumulativeIncidenceAnalysis~. We start out by creating the
instance:

#+BEGIN_SRC R :exports code
cif_analysis <- CumulativeIncidenceAnalysis$new()
#+END_SRC

#+RESULTS:

Compared to the previous guide, we are going to run the analysis twice, once with the
whole survival dataset and once with a subset that only contains individuals that have
atleast one affected relative. The lifetime prevalence for these two cohorts are then
compared when calculating heritability. To do that, we need to add the subset onto the
survival dataset and then add a grouping column for the two sets:

#+BEGIN_SRC R :exports both :results output
tte <- tte |>
  mutate(cohort = "all") |>
  union_all(
    tte |>
      filter(diagnosed_relatives > 0) |>
      mutate(cohort = "affected_relatives")
  )
#+END_SRC

#+RESULTS:

When running the analysis separately on different groups, you use the ~run~
as before, but you also supply the column that contains the group values:

#+BEGIN_SRC R :exports code :results output
estimates <- cif_analysis$run(
  tte = tte,
  earliest_onset_age = earliest_onset_age,
  group_columns = list("cohort")
)
#+END_SRC

#+RESULTS:

Note: You can use multiple columns to group the results on. The analysis will then
produce results for each permutation of the columns available values.

** Heritability

After producing the risk estimates, we can calculate the heritability.
To do that we need a dataset that contains the last risk estimate
(highest value of ~time~ column) that exists in both cohorts.
To make sure we only pick the last risk estimates that exists in both cohorts,
we will first split the risk estimates into two separate datasets, one for each cohort:

#+BEGIN_SRC R :exports both :results output
cohort1 <- estimates |> filter(cohort == "all")
cohort2 <- estimates |> filter(cohort == "affected_relatives")
#+END_SRC

#+RESULTS:

Then we join the two datasets with an inner join. An inner join will only keep rows that
have matches in both datasets, all other rows are removed:

#+BEGIN_SRC R :exports both :results output
combined <- cohort1 |>
  inner_join(cohort2, by = join_by(time)) |>
  rename(
    c1_estimate = estimate.x,
    c1_cases    = cases.x,
    c2_estimate = estimate.y,
    c2_cases    = cases.y
  ) |>
  select(time, c1_estimate, c1_cases, c2_estimate, c2_cases)
#+END_SRC

#+RESULTS:

Now we have a combined dataset where each row represents risk estimates from both cohorts.
Since we are only interest in the last risk estimate, we need to sort the dataset on
the ~time~ (largest first) column and pick the first row:

#+BEGIN_SRC R :exports both :results output
combined <- combined |>
  arrange(desc(time)) |>
  filter(row_number() == 1) |>
  as.data.table()

print(combined)
#+END_SRC

#+RESULTS:
: time c1_estimate c1_cases c2_estimate c2_cases
: 1:   45   0.1055564     1942   0.2092641      874

Now we can simply run the heritability calculation by passing in the the estimate and
cases columns from the combined dataset along with the relationship kind:

#+BEGIN_SRC R :exports both :results output
h2_analysis <- HeritabilityAnalysis$new()

results <- h2_analysis$run(
  relationship_kind = relationship_kind,
  estimates         = combined
) |> select(time, estimate, se, l95, u95) # We only need the new columns added by the h2 analysis
#+END_SRC

#+RESULTS:

And here's what the results look like:

#+BEGIN_SRC R :exports both :results output
print(results)
#+END_SRC

#+RESULTS:
:    time  estimate         se       l95       u95
: 1:   45 0.5392606 0.02765919 0.4850486 0.5934727

All done!

The results are joined with the input risk estimates, so when you analyse multiple rows,
you can see which risk estimates produced which heritabilities. In the pre-assembled
guide [[./guide-yob.org][estimate heritability of disorder stratified by year of birth]] where we get one
heritability per birth year, you can see why this is needed.
