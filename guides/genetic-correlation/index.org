#+TITLE: Genetic correlation
#+AUTHOR: Richard Zetterberg <richard.zetterberg@regionh.dk>

Genetic correlation is produced by the [[https://r6.r-lib.org/articles/Introduction.html][R6 class]] ~GeneticCorrelationAnalysis~ included in this package.
You can learn how to use this class, either by reading the reference documentation below or read
[[./guide.org][this guide that shows you how it is used, step-by-step with code examples.]]

* Functional reference

To use the class, you need to create an instance first:

#+BEGIN_SRC R
analysis <- GeneticCorrelationAnalysis$new()
#+END_SRC

Then you use that instance to run one of the 3 functions it exposes:

- run :: Produces the genetic correlations from the cumulative incidences and heritabilities of two disorders
- run_meta :: Meta analyses stratified heritability estimates, outputs fixed and random model results

** Using function ~run~

This function accepts 2 named arguments:

- relationship_kind (required) :: The relationship kind between the sample and their relatives (full sibling, parent-offspring, etc.)
- estimates (required) :: A ~data.table~ that contains the cumulative incidences and heritabilities of two disorders,
  that must contain the following columns:

  |----------------------+-----------------------------------------------------------------|
  | Column name          | Description                                                     |
  |----------------------+-----------------------------------------------------------------|
  | *re_d1_c1_estimates* | Floating point number, disorder 1, cohort 1 risk estimate       |
  | *re_d1_c1_cases*     | Natural number, disorder 1, cohort 1 cumulative amount of cases |
  | *re_d1_c3_estimates* | Floating point number, disorder 1, cohort 3 risk estimate       |
  | *re_d1_c3_cases*     | Natural number, disorder 1, cohort 3 cumulative amount of cases |
  | *re_d2_c1_estimates* | Floating point number, disorder 2, cohort 1 risk estimate       |
  | *re_d2_c1_cases*     | Natural number, disorder 2, cohort 1 cumulative amount of cases |
  | *h2_d1*              | Floating point number, disorder 1 heritability estimate         |
  | *h2_d2*              | Floating point number, disorder 2 heritability estimate         |
  |----------------------+-----------------------------------------------------------------|

  This ~data.table~ is allowed to contain any number of extra columns, such as when the reults are stratified and there are
  columns that contains the different group values. An example of this is when the estimates are stratified by
  birth year, then you would have a column that contains the birth years. Those extra columns are untouched and will be
  kept in the results.

Note: This function also accepts stratified cumulative incidence estimates, where there are one row per stratified group.
